# AGENTS.md

## Commands

- Use npm; this repo has `package-lock.json` and no pnpm/yarn lockfile.
- `npm run dev` starts the Vite dev server.
- `npm run build` is the only configured verification command; it runs `tsc` first, then `vite build`.
- `npm run preview` serves the production build.
- There are no configured test, lint, or formatter scripts. Do not invent `npm test` or `npm run lint` as required checks.

## Architecture

- Single-package React 19 + Vite + TypeScript app; entrypoint is `src/main.tsx`, routing is in `src/app/App.tsx`.
- Routes are `/` and `/tools/:toolId`; each tool is lazy-loaded from `src/tools/registry.ts`.
- Add or rename tools in `src/tools/registry.ts`; search, grouping, route loading, favorites, and recents all depend on registry `id`, `category`, `keywords`, and `load`.
- Tool UI should use existing shared pieces such as `Panel`, `CopyButton`, and the `tool-workspace` / `tool-section` CSS classes instead of adding a new layout system.
- UI copy is centralized in `src/app/providers/LocaleProvider.tsx`; add both `zh-CN` and `en-US` strings for visible text.

## Local-First Constraints

- The product promise is browser-local processing. Do not add server calls for normal tool execution.
- Favorites, recents, locale, and theme are stored in `localStorage`; storage helpers intentionally swallow failures for private/restricted contexts.
- Sensitive tools should make local processing clear in the UI.
- HTML preview must stay sandboxed and must not add `allow-same-origin`; current iframes use `sandbox="allow-scripts allow-modals"` and `referrerPolicy="no-referrer"`.

## Generated Data

- `src/tools/generate/us-addresses.generated.ts` is generated by `scripts/build-us-address-pool.mjs` from OpenAddresses data.
- There is no package script for that generator; run it directly with `node scripts/build-us-address-pool.mjs` only when intentionally refreshing the address pool.
- `src/tools/security/wordlists.generated.ts` is a generated wordlist source; avoid hand-editing unless the task is explicitly about that generated data.

## Design And Copy

- Treat `DESIGN.md` as the design source of truth.
- Chinese is the primary interface language; English supports scanning and utility naming.
- Preserve the calm premium utility style: warm ivory/dark graphite themes, restrained sage/brown accent, hairlines/inset detail, minimal shadows.
- Avoid terminal green, cyberpunk neon, AI-purple gradients, generic admin dashboards, and equal three-card marketing rows.
- Do not use `h-screen`; use `min-height: 100dvh` where full-height behavior is needed.
- Multi-column tool layouts should collapse for small screens; existing CSS collapses `.tool-workspace.two-col` and `.wide` at `1040px`.
- Motion should only animate `transform` and `opacity`, use the repo `--ease` cubic-bezier, and respect `prefers-reduced-motion`.
